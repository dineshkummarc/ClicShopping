# Main analysis prompt
unified_analyzer_prompt = Analyze this user query, translate it, and break down complex analytic questions. Return a JSON object with the following fields:\n\n1. "language": ISO 639-1 language code (2 letters: en, fr, ja, zh, es, de, ar, it, pt, ru, ko, nl, pl, tr, sv, da, fi, no, cs, hu, ro, el, he, th, vi, id, ms, hi, bn, ta, te, mr, ur, fa, uk, bg, sr, hr, sk, sl, lt, lv, et, is, ga, mt, cy, eu, ca, gl, sq, mk, bs, ka, hy, az, kk, uz, ky, tg, tk, mn, ne, si, km, lo, my, am, ti, sw, zu, xh, af, so, ha, yo, ig, rw, sn, st, tn, ts, ss, ve, nr, etc.)\n\n2. "translated_query": Query translated to English (or original if already English)\n\n3. "intent_type": One of:\n   - "analytics": Queries about data, metrics, counts, statistics, numbers, reports\n     Examples: "How many products?", "Sales today", "Total revenue", "Stock levels", "Top 10 customers"\n   - "semantic": Queries about knowledge, explanations, policies, procedures, how-to\n     Examples: "What is the return policy?", "How does delivery work?", "Explain checkout process"\n   - "hybrid": Queries combining both analytics and semantic aspects\n     Examples: "Show me sales data and explain why it's low", "How many orders and what's the refund policy?"\n\n4. "entity_type": A list of one or more primary entities involved (e.g., ["product", "order", "customer", "general"]). Use "general" for global metrics without specific entities.\n\n5. "time_constraint": One of: "comparison", "relative_period", "specific_date", "none"\n\n6. "status_keywords": A list of status keywords detected (e.g., ["active", "pending"]). Use empty array [] if no status is relevant.\n\n7. "sub_queries": A list of individual, separated sub-queries if a multi-query detection is warranted. Use empty array [] for single queries.\n\n8. "confidence": Confidence score between 0.0 and 1.0 (how certain you are about the intent classification)\n\nQuery: "%s"\n\nReturn ONLY valid JSON, no additional text or explanation.
