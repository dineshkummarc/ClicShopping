# Prompt d'analyse principal
unified_analyzer_prompt = Analysez cette requête utilisateur, traduisez-la et décomposez les questions analytiques complexes. Retournez un objet JSON avec les champs suivants :\n\n1. "language": Code de langue ISO 639-1 (2 lettres : en, fr, ja, zh, es, de, ar, it, pt, ru, ko, nl, pl, tr, sv, da, fi, no, cs, hu, ro, el, he, th, vi, id, ms, hi, bn, ta, te, mr, ur, fa, uk, bg, sr, hr, sk, sl, lt, lv, et, is, ga, mt, cy, eu, ca, gl, sq, mk, bs, ka, hy, az, kk, uz, ky, tg, tk, mn, ne, si, km, lo, my, am, ti, sw, zu, xh, af, so, ha, yo, ig, rw, sn, st, tn, ts, ss, ve, nr, etc.)\n\n2. "translated_query": Requête traduite en anglais (ou originale si déjà en anglais)\n\n3. "intent_type": Un parmi :\n   - "analytics": Requêtes sur les données, métriques, comptages, statistiques, nombres, rapports\n     Exemples : "Combien de produits ?", "Ventes aujourd'hui", "Revenu total", "Niveaux de stock", "Top 10 clients"\n   - "semantic": Requêtes sur les connaissances, explications, politiques, procédures, comment faire\n     Exemples : "Quelle est la politique de retour ?", "Comment fonctionne la livraison ?", "Expliquer le processus de paiement"\n   - "hybrid": Requêtes combinant aspects analytiques et sémantiques\n     Exemples : "Montrez-moi les données de ventes et expliquez pourquoi elles sont basses", "Combien de commandes et quelle est la politique de remboursement ?"\n\n4. "entity_type": Une liste d'une ou plusieurs entités principales impliquées (ex : ["product", "order", "customer", "general"]). Utilisez "general" pour les métriques globales sans entités spécifiques.\n\n5. "time_constraint": Un parmi : "comparison", "relative_period", "specific_date", "none"\n\n6. "status_keywords": Une liste de mots-clés de statut détectés (ex : ["active", "pending"]). Utilisez un tableau vide [] si aucun statut n'est pertinent.\n\n7. "sub_queries": Une liste de sous-requêtes individuelles séparées si une détection multi-requête est justifiée. Utilisez un tableau vide [] pour les requêtes simples.\n\n8. "confidence": Score de confiance entre 0.0 et 1.0 (à quel point vous êtes certain de la classification d'intention)\n\nRequête : "%s"\n\nRetournez UNIQUEMENT du JSON valide, sans texte ou explication supplémentaire.
